# Peer Tutoring — 프론트엔드 Task 정리

> 작성 기준: 2026-05  
> 기술 스택: Next.js + TypeScript + Supabase

---

## 1. 전제 조건 및 역할 분리

| 역할 | 담당 범위                                                                                                         |
| ---- | ----------------------------------------------------------------------------------------------------------------- |
| 총괄 | 환경 세팅, 폴더 구조, Supabase 연결, Auth, DB fetch/INSERT/UPDATE, 비즈니스 로직, 팀원 결과물 검토 후 백엔드 연결 |
| 팀원 | props를 받아서 UI를 그리는 순수 컴포넌트 작성, 더미 데이터로 레이아웃 완성                                        |

**핵심 원칙:** 팀원 task는 Supabase를 몰라도 시작할 수 있는 수준으로 설계한다.
모든 데이터 연결은 총괄이 팀원 UI 완성 후 주입한다.

---

## 2. 페이지 위계 구조

```
/login
  └── / (홈 — 라우팅 허브)
        ├── /questions          (질문 목록)
        │     └── /questions/[id]  (질문 상세)
        ├── /questions/new      (질문 작성)
        ├── /ranking            (주간 랭킹, P3)
        └── /profile/[id]       (프로필, 껍데기 P1 / 완성 P3)
```

**공통 요소 (모든 페이지에 포함)**

- GNB (글로벌 네비게이션 바)
- Auth Guard (세션 없으면 /login 리다이렉트)
- LoadingSpinner / ErrorMessage / Toast

**Phase 2에서 기존 페이지 내부에 삽입되는 UI**

- 채택 버튼 → `/questions/[id]` 내부
- 이미지 업로드 → 질문·답변 폼 내부
- 알림 드롭다운 → GNB 내부
- 포인트 내역 탭 → `/profile` 내부

**Phase 4에서 추가**

- 출석체크 위젯 → 홈 내부
- 즐겨찾기 탭 → `/profile` 내부

## 6. 컴포넌트 작성 규칙 (팀원 공통)

1. **더미 데이터는 파일 상단에 하드코딩** — `const DUMMY_QUESTIONS = [...]`
2. **모든 컴포넌트는 props 타입을 TypeScript interface로 정의** — 총괄이 interface 초안 제공 예정
3. **스타일은 Tailwind CSS 클래스 사용** — 총괄이 디자인 토큰(색상, 간격) 가이드 제공
4. **클릭 핸들러가 필요한 버튼은 `onClick?: () => void` props로 뚫어두기** — 로직은 총괄이 주입
5. **컴포넌트 파일 위치** — `src/components/페이지명/컴포넌트명.tsx`
