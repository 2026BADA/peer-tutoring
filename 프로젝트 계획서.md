# peer tutoring 프로젝트 계획서

peer tutoring은 북일고 내에서 모르는 것을 질의응답할 수 있는 플랫폼입니다.

---

## 1. 프로젝트 개요

### 1.1 서비스 소개

학생·전문가 누구나 과목별로 질문을 올리고 답변을 주고받을 수 있는 지식 공유 커뮤니티 플랫폼입니다. 네이버 지식인의 채택 시스템을 기반으로, 포인트·티어·뱃지 등 게임화(Gamification) 요소를 더해 지속적인 참여를 유도합니다.

### 1.2 핵심 가치

| 질문과 답변                      | 성장과 보상                  | 지식 공유                        |
| -------------------------------- | ---------------------------- | -------------------------------- |
| 채택 시스템으로 검증된 정보 제공 | 포인트·티어·뱃지로 기여 인정 | 과목별 분류로 체계적인 지식 축적 |

---

## 2. 기능 정의

### 2.1 질문 & 답변 시스템

서비스의 핵심 기능으로, 모든 활동의 중심이 됩니다.

**질문 작성**

- 과목(카테고리) 선택 필수 — 수학, 과학, 영어, 사회 등
- 텍스트 + 이미지 첨부 가능
- 질문 작성 시 포인트 소비
- 답변이 달린 후에는 질문 삭제 불가

**답변 작성**

- 텍스트 + 이미지 첨부 가능
- 답변 작성만으로도 포인트 획득
- 질문자가 답변 채택 가능 (1개 채택 시 해당 질문 종료)
- 채택 시 답변자에게 추가 포인트 지급

**자동 채택 규칙**

질문자가 일정 기간(예: 7일) 내에 채택하지 않으면 최다 추천 답변이 자동 채택됩니다. 추천 수가 동일하면 가장 먼저 달린 답변이 채택됩니다.

**정렬 & 탐색**

- 기본 정렬: 최신순
- 추후 추가: 인기순, 미채택순, 내 과목순
- 과목별 필터링

---

### 2.2 포인트 경제 시스템

포인트는 플랫폼 내 유일한 화폐이며, 획득·소비·순환 구조를 통해 참여를 유도합니다.

| 활동             | 포인트 변동 | 비고                         |
| ---------------- | :---------: | ---------------------------- |
| 질문 작성        |    −10 P    | 소비                         |
| 답변 작성        |    +5 P     | 획득                         |
| 답변 채택됨      |    +30 P    | 획득 (질문자가 채택)         |
| 자동 채택됨      |    +15 P    | 획득 (자동 처리)             |
| 질문에 추천 받음 |    +2 P     | 획득                         |
| 출석 체크        |  +3 P / 일  | 획득                         |
| 포인트 사용처    |     −P      | 추후 기획 (프로필 꾸미기 등) |

> ※ 위 수치는 초기 기획안이며, 밸런스 테스트 후 조정 예정입니다.

---

### 2.3 티어 & 뱃지 시스템

**티어 (포인트 기반 자동 승급)**

|   티어    | 이름       |   필요 포인트   | 혜택         |
| :-------: | ---------- | :-------------: | ------------ |
| 🥉 Tier 5 | 브론즈     |    0 ~ 499 P    | 기본         |
| 🥈 Tier 4 | 실버       |  500 ~ 1,499 P  | 질문 추가 +1 |
| 🥇 Tier 3 | 골드       | 1,500 ~ 3,999 P | 질문 추가 +2 |
| 💎 Tier 2 | 플래티넘   | 4,000 ~ 9,999 P | 질문 추가 +3 |
| 👑 Tier 1 | 다이아몬드 |  10,000 P 이상  | 모든 혜택    |

> 승급 포인트 책정 잘못된 듯 함.

**뱃지 (도전과제 달성 기반)**

| 뱃지        | 달성 조건                    |
| ----------- | ---------------------------- |
| 수학 고수   | 수학 카테고리 답변 10회 채택 |
| 지식 나눔이 | 총 답변 수 50개 달성         |
| 연속 출석   | 7일 연속 로그인              |
| 스타터      | 첫 번째 질문 작성            |
| 채택왕      | 한 주 채택 수 1위            |

> 추가 도전과제는 운영 중 지속 확장 예정

---

### 2.4 주간 랭킹

- 집계 기준: 해당 주 채택 수 + (획득 포인트 × 0.1) 복합 점수
- 매주 월요일 00:00 초기화
- 상위 10명 공개 노출
- 1위 달성 시 "이주의 지식인" 뱃지 지급

---

### 2.5 사용자 프로필

- 보유 티어 및 뱃지 목록
- 누적 통계: 질문 수, 답변 수, 채택률, 총 포인트
- 내 질문 모아보기
- 내 답변 모아보기
- 즐겨찾기한 질문 목록

---

### 2.6 알림 시스템

- 내 질문에 답변이 달렸을 때
- 내 답변이 채택되었을 때
- 자동 채택이 발생했을 때
- 뱃지를 새로 획득했을 때
- 티어가 변경되었을 때

## 6. 미결 사항 (결정 필요)

### 우선 결정 필요

- **포인트 수치 밸런스** — 질문 소비량 / 답변·채택 획득량 비율 확정
- **자동 채택 대기 기간** — 7일? 14일? (너무 짧으면 불만, 너무 길면 방치)
- **포인트 소비처 기획** — 프로필 꾸미기 아이템, 질문 상단 노출 부스터 등 검토

### 추후 결정 가능

- 댓글 / 대댓글 기능 도입 여부 및 시점
- 신고 / 제재 시스템 설계
- 비로그인 사용자 질문 열람 허용 여부 (SEO 관점에서 허용 권장)
- 인기순 정렬 알고리즘 정의 방식

## 7. 보완 제안

### 7.1 신고 & 제재 시스템

MVP 출시 시점부터 최소한의 신고 기능을 준비해야 합니다. 스팸·욕설 처리 없이는 커뮤니티 품질이 빠르게 하락합니다.

- 신고 버튼 (질문 / 답변 단위)
- 누적 신고 N회 → 자동 블라인드 처리
- 관리자 대시보드 (간단한 어드민)

## 8. 용어 정의

| 용어      | 정의                                                                              |
| --------- | --------------------------------------------------------------------------------- |
| 채택      | 질문자가 답변 중 하나를 정답으로 선택하는 행위. 채택 시 질문 상태가 "종료"로 변경 |
| 자동 채택 | 채택 기한 내 질문자가 채택하지 않을 경우, 시스템이 최다 추천 답변을 채택 처리     |
| 포인트(P) | 플랫폼 내 모든 활동의 기준이 되는 가상 화폐. 외부 환전 불가                       |
| 티어      | 누적 포인트를 기반으로 자동 부여되는 등급 (브론즈~다이아몬드)                     |
| 뱃지      | 특정 도전과제 달성 시 부여되는 칭호. 티어와 독립적으로 운영                       |
| 도전과제  | 뱃지 획득 조건이 되는 특정 행동 기준 (예: 답변 채택 10회)                         |
