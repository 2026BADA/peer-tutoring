# Peer Tutoring 팀원 온보딩 가이드

> 협업 가이드 문서

---

## 1. 시작 전 준비

아래 두 가지가 설치되어 있어야 한다.

- [Node.js](https://nodejs.org/ko) 설치 (LTS 버전)
- [VS Code](https://code.visualstudio.com/) 설치

---

## 2. 프로젝트 받기

터미널을 열고 아래 순서대로 입력한다.

```bash
# 1. 프로젝트 받기
git clone https://github.com/2026BADA/peer-tutoring.git

# 2. 폴더 들어가기
cd peer-tutoring

# 3. 필요한 패키지 설치
npm install

# 4. 내 브랜치로 이동 (없으면 만들기)
git checkout -b 이름

# 5. 개발 서버 실행
npm run dev
```

브라우저에서 `http://localhost:3000` 열었을 때 화면이 뜨면 성공이에요!

---

## 3. 폴더 구조

```
src/
├── app/                  # 페이지 라우팅
├── components/           # 컴포넌트 모아놓은 폴더
│   ├── ui/               # 공통 컴포넌트 (버튼, 카드 등)
│   ├── questions/        # 질문 관련 컴포넌트
│   └── profile/          # 프로필 관련 컴포넌트
└── lib/                  # Supabase 설정 (건드리지 않아도 됨)
```

**컴포넌트는 `src/components/` 안에만 만들도록 하자.**

---

## 4. 컴포넌트 작성 규칙

### 파일 위치

```
src/components/페이지명/컴포넌트명.tsx
```

예시:

```
src/components/questions/QuestionCard.tsx
src/components/ui/LoadingSpinner.tsx
```

### 더미 데이터 사용법

실제 데이터 연결은 나주엥, 지금은 파일 맨 위에 가짜 데이터를 하드코딩해서 쓴다.

```tsx
// 파일 맨 위에 더미 데이터 선언
const DUMMY_QUESTIONS = [
    { id: 1, title: "수학 질문이에요", authorName: "홍길동" },
    { id: 2, title: "영어 질문이에요", authorName: "김철수" },
];

export default function QuestionList() {
    return (
        <div>
            {DUMMY_QUESTIONS.map((q) => (
                <div key={q.id}>
                    <h2>{q.title}</h2>
                    <p>{q.authorName}</p>
                </div>
            ))}
        </div>
    );
}
```

## 5. 작업 후 PR 올리는 법

작업이 끝나면 아래 순서로 올린다.

```bash
# 1. 변경된 파일 전부 추가
git add .

# 2. 커밋 (따옴표 안에 작업 내용 써요)
git commit -m "QuestionCard 컴포넌트 작성"

# 3. 내 브랜치에 올리기
git push origin 이름
```

그 다음 GitHub에서 **Pull Request** 버튼 누르고 팀장한테 알린다.
